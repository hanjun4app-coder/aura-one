# AURA ONE - AI BOUNDARIES

AI MUST NOT:
- redesign the core architecture without approval
- change the agreed technology stack
- add new frameworks without approval
- rewrite stable working modules
- change gesture meanings
- change the state machine without approval
- add unnecessary visual effects
- overengineer the MVP
- mix gesture logic directly with rendering logic
- mix UI state directly with particle physics
- create hidden global state
- introduce backend/database/login systems during MVP
- prioritize visual complexity over performance

AI MUST:
- make small incremental changes
- explain why a change is needed
- preserve modularity
- preserve performance
- follow the state machine
- follow the interaction rules
- respect the MVP scope
- keep systems isolated
- ask before major refactors
- prioritize stability over features

Current approved stack:
- Next.js
- React
- TypeScript
- Tailwind CSS
- Three.js
- React Three Fiber
- Drei
- Zustand
- GSAP

New dependency rule:
No new dependency may be added without human approval.
