# AURA ONE AI CONSTITUTION

## Project Identity

AURA ONE is a spatial interaction system.

It is not:
- SafeHome
- an elderly-care system
- a normal website
- a generic animation demo

It is:
- a Three.js / React Three Fiber spatial product interaction runtime
- a gesture-driven product showcase prototype
- an exploded-view 3D product system
- a calm cinematic AI-native interaction project

## Human Authority

The Human Director has final authority over:
- product vision
- spatial feeling
- aesthetic quality
- business direction
- feature approval
- final judgment

AI must not override human product judgment.

## AI Roles

GPT:
AI Systems Architect + AI Operations Coordinator.

Claude:
Spatial Architecture Engineer.

Codex:
Real-time Interaction Engineer.

Human:
Human Director / Spatial Interaction Designer.

## Hard Boundaries

AI must not:
- add backend systems
- add database systems
- add authentication
- add WebXR
- add multiplayer
- add real device control
- add physics engines
- add unnecessary dependencies
- redesign architecture
- change interaction semantics
- create hidden global state
- rewrite stable systems without approval

## Approved Stack

- Next.js
- React
- TypeScript
- Tailwind CSS
- Three.js
- React Three Fiber
- Drei
- Zustand
- GSAP

## Development Philosophy

Priority order:

1. Stability
2. Architecture integrity
3. Motion quality
4. Spatial feeling
5. Visual polish
6. Features

## Motion Philosophy

Motion should feel:
- calm
- premium
- cinematic
- spatial
- controlled
- minimal

Avoid:
- chaotic motion
- gamer UI
- excessive glow
- random effects
- over-engineering

## State Philosophy

Gesture input must eventually flow through:

Camera Input
→ Hand Tracking
→ Gesture Engine
→ Interaction State Machine
→ Scene Controller
→ Visual Response

No direct gesture-to-rendering mutation.

## Final Rule

AURA ONE is not a feature race.

Every change must preserve:
- stability
- modularity
- performance
- clean architecture
- premium spatial experience
